# Integration Tests Contract

`tests/integration` owns cross-service Overrid scenarios and scenario manifests for the Integration Test Harness.

Tracked source contracts:

- `scenarios/`: source-controlled scenario manifests and scenario notes.

Ignored local-only markers:

- `artifacts/`: redacted logs, CLI output, API payload envelopes, stack health exports, fixture outputs, and reproduction bundles generated by local runs.

Rules:

- Scenarios must be phase-gated and fixture-backed.
- Artifacts must be secret-free and ignored unless a later acceptance path explicitly promotes a redacted fixture.
- Integration tests must not become runtime dependencies of services or packages.
